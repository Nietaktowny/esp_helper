10/*
11 * arena_init - Initializes an arena with memory from the heap. The arena will
12 * contain `count' elements, all of size `size'. All allocations
13 * made by the arena will be `size' bytes long. When all of the
14 * arena's elements have been allocated, arena_alloc returns NULL.
15 *
16 * arena_init_ - Initializes an arena using pre-allocated memory at `mem', and
17 * of length `len'. The first two parameters are the same as
18 * arena_init. This version lets you allocate an arena wherever
19 * you want; whether it be inside another arena, on the stack, in
20 * a memory map, or where ever your heart desires.
21 *
22 * If you use this version of arena_init, DO NOT call
23 * arena_destroy. It will attempt to deallocate the underlying
24 * memory. You must do that manually, by whatever method it was
25 * originally allocated.
26 *
27 * When using arena_init_, len must be at least
28 * max(size, 2*sizeof(size_t))*count + 7*sizeof(size_t) so
29 * that the arena header can be placed directly next to the raw
30 * memory, instead of being heap allocated.
31 */
